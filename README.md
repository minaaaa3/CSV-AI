🚀 Sales Insight Workforce (Prototype)
CSV形式の売上データを自動解析し、ビジネスボトルネックの特定から、具体的な改善施策（ガントチャート付）の提案までを一気通貫で実行するAIエージェントです。
「AIが、ソフトウェアの裏側で、自律的に業務を遂行する（Workforce）」という思想を具現化するためのプロトタイプとして開発しました。

🛠 解決する課題 (The Problem)
分析の属人化: 売上データの変化に気づく、あるいはその要因を特定する作業が特定の分析担当者に依存している。

「分析」と「実行」の分断: データ分析結果（KPIの変動）が出ても、それを具体的な現場の施策やスケジュールに落とし込むまでに時間差と工数がかかっている。

LLMの非構造化問題: 通常のチャットAIでは出力が安定せず、後続の業務システムとの連携が困難。

✨ 主な機能 (Key Features)
1. Pydanticによる「壊れない」構造化出力
GeminiのStructured Output機能を活用。独自のデータモデル（models.py）により、AIの回答を厳格なJSON形式で取得します。これにより、ビジネスインサイトの抽出ミスを防ぎ、プログラムによる後続処理（KPI表示やファイル保存）の安定性を担保しています。

2. コンテキストを汲み取った施策立案
単なる要約ではなく、特定されたボトルネック（例：特定エリアの売上急減）に基づき、**「誰が・何を・いつまでに」**すべきかを提案します。

3. Mermaid記法によるスケジュールの可視化
提案された施策は、GitHubやNotionでそのまま図解できる**Mermaid (Gantt形式)**で出力されます。

💡 こだわり・工夫した点 (Engineering Excellence)
1. レジリエンス（回復性）のある設計
開発過程で直面した最新モデル（Gemini 2.0/3.x）のAPIクォータ制限に対し、安定版モデルへのフォールバック（切り替え）ロジックを実装。また、.env読み込み時のパス問題をos.pathによる動的解決で突破するなど、環境に依存しない安定稼働を追求しました。

2. セキュリティと構成管理
APIキーなどの機密情報は.envで完全に分離し、.gitignoreと.env.exampleを適切に運用することで、チーム開発やデプロイを想定したセキュアな構成管理を行っています。
